# PMD CLI参考

概要：命令行的参数选项、输出格式以及支持的语言方面的详细介绍。

# PMD命令选项参数

可以使用 --help选项查看所有选项参数。

选项描述默认值应用于
--rulesets <refs>
-R <refs>		必选。规则集xml文件路径。可以是应用classpath下的文件，可以是操作系统的本地路径，或者一个URL。多个值使用逗号分隔。
--dir <path>
-d <path>		源文件路径，或者包含源文件压缩包(.zip)、jar包的目录。--dir/--file-list/--uri三个参数必须使用其中一个。目前测试只分析源文件，不能分析class文件。
--format <format>
-f <format>		指定输出报告的格式。所有格式open in new windowtext
--aux-classpath <cp>指定源码依赖库的路径。这用于解析源文件中的类型。平台特定的路径分隔符（在Linux上为“:”，在Windows上为“;”）用于分隔条目。或者，您可以指定一个指向包含连续行上的路径元素的文本文件的单个文件URL。Java
--benchmark
-b		开启基准模式。输出基准报告。
--cache <filepath>定义增量分析时缓存存放目录。完整的文件路径，而不是文件存放目录。如果文件不存在会自动创建。运行时触发更新规则时，会覆盖该文件。推荐配置
--debug
--verbose
-D
-v		Debug参数。输出更多调试日志。参考Loggingopen in new window。
--encoding <charset>
-e <charset>		定义PMD读取源文件的字符集类型。支持的字符集同“java.nio.charset.Charset”。UTF-8
--[no-]fail-on-violation定义发现问题是是否以非 零状态退出分析程序。默认是以状态4退出。如果配置--no-fail-on-violation将以状态0退出，并且输出检测报告。
--file-list <filepath>配置文件列表文件的路径。文件中一行一个文件。参数--dir ,--file-list,--uri参数三选一。
--force-language <lang>强制统一所有输入文件语言类型，忽略文件名。配置此选项后，不在根据文件扩展名判断文件语言。pmd将所有文件使用配置的语言进行处理。处理错误或者无法处理的文件将被忽略。假如不适用语言的默认版本，使用--use-verison可以指定。一个分析xml文件扩展名不是.xml的例子open in new window。
--ignore-list <filepath>配置一个文件。文件里面配置需要忽略的文件，一行一个。这个选项配置将忽略--dir，--file-list，--uri引入的分析文件。
--help
-h		显示指令选项信息
--use-version <lang-version>定义处理当指定语言类型时，使用的语言版本。配置值需要符合language-version格式。可以指定多个语言的版本。此配置不会影响文件语言类型的判断，只有在判断出语言类型之后，才会按照配置语言版本进一步分析。语言检测只受文件扩展名和--force-language选项影响。 支持的语言open in new window
-language <lang>
-l <lang>		已废弃。使用--use-version替代。
--minimum-priority <priority>规则优先级阈值。低于此处配置的优先级问题将不被检测。有效值(不区分大小写)High、Medium_High、Medium、Medium_Low、Low，或者数字1(High)到5(Low)。参数覆盖说明Configuring rulesopen in new windowLow
--no-cache显示关闭增量分析功能。改配置将使--cache配置失效。
--[no-]progress开启或者禁用分析过程中的进度条显示。ie上默认开启
--property <name>=<value>
-P <name>=<value>		定义报告生成器参数。该参数可以多次定义。使用--help查看所有可定义的参数。
--relativize-paths-with <path>
-z <path>		只生成指定目录下文件的检测报告。该参数可以重复配置以配置多个目录。
--report-file <path>
-r <path>		定义报告输出文件，如果文件没有将被创建。如果该参数未定义，报告将输出到标准路径。
--show-suppressed报告中显示被忽略的检测问题。
--suppress-marker <marker>定义需要PMD忽略该行的注释标志。NOPMD
--threads <num>
-t <num>		PMD分析时开启的进程数。设置为0代表禁止多线程分析。1
-uri <uri>
-u <uri>		源文件数据库URI。--dir,--file-list,--uri三个参数必须配置一个。PLSQL

# Java 运行时选项

PMD是在java虚拟机上运行的。有时需要设置一些java运行时参数。

# PMD_JAVA_OPTS

linux

~ $ export PMD_JAVA_OPTS="--enable-preview"
~ $ pmd check -d src/main/java/ -f text -R rulesets/java/quickstart.xml

windows

C:\> set "PMD_JAVA_OPTS=--enable-preview"
C:\> pmd.bat check -d src\main\java\ -f text -R rulesets/java/quickstart.xml

# classpath

​ 在定制规则打成jar包时，可以通过防止到PMD指定的lib目录下或者放在任意位置，然后配置该位置目录到classpath。

​ 配置classpath指定规则扩展包。

linux

~ $ export CLASSPATH=custom-rule-example.jar
~ $ pmd check -d ../../../src/main/java/ -f text -R myrule.xml

windows

C:\> set CLASSPATH=custom-rule-example.jar
C:\> pmd.bat check -d ..\..\..\src\main\java\ -f text -R myrule.xml

# PMD退出状态

​ 默认退出状态为4.

0没有发现问题，正常退出。
1PMD运行异常。
2使用错误。指令参数无效或缺失。
4至少发现一个问题出发设置了--no-fail-on-violation。

# 日志

​ pmd日志使用slf4j,slf4j的实现为slf4j-simple。日志信息通过System.err输出。所以报告信息一般都要配置输出到文件，否则就会输出到控制台。

​ slf4j-simple的配置文件为conf/simplelogger.properties。--debug参数将日志级别设置为debug。

# 支持的语言

​ PMD会根据文件的扩展名自动识别文件的语言，比如java。PMD可以支持语言的不同版本。未配置语言版本时，将使用默认的版本。默认版本一般都是最新的非预览版。如果需要使用更旧的版本使用--use-version选项配置。

​ 语言的不同版本会影响到规则集，因为有些规则只适用于特定版本的语言。这种规则使用参数minimumLanguageVersion和maximumLanguageVersion来确定应用的版本。但是大部分规则是不和语言版本关联的。

​ PMD 命令--help可以查询出PMD支持的语言版本。

​ 指定语言版本的例子

~ $ pmd check -d src/main/java -f text -R rulesets/java/quickstart.xml --use-version java-1.8

C:\> pmd.bat check -d src\main\java -f text -R rulesets/java/quickstart.xml --use-version java-1.8

# 一些例子

# 分析其他格式xml

​ 如果xml的源文件扩展名没有使用xml，可以使用--force-languge参数来指定源文件语言为xml。

~ $ pmd check -d src/xml-file.ext -f text -R ruleset.xml --force-language xml

C:\> pmd.bat check -d src\xml-file.ext -f text -R ruleset.xml --force-language xml

​ -d 参数值也可以配置成一个目录。分析目录下所有文件时，单个文件无法处理时，将不影响PMD的正常运行。

~ $ pmd check -d src/ -f text -R ruleset.xml --force-language xml

C:\> pmd.bat check -d src\ -f text -R ruleset.xml --force-language xml

​ 限定源文件的扩展名

~ $ find src/ -name "*.ext" > filelist.txt
~ $ pmd check --file-list filelist.txt -f text -R ruleset.xml --force-language xml

C:\> for /r src/ %i in (*.ext) do echo %i >> filelist.txt
C:\> pmd.bat check --file-list filelist.txt -f text -R ruleset.xml --force-language xml
